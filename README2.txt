Test Cases

Test Case 1: Voting up on a software tool

Description: Ensure that when a user votes up, the upvote count increases by one.
Expected Result: The tool’s upvote count is incremented by one, and the new count displays immediately.

Test Case 2: Voting down on a software tool

Description: Ensure that voting down decreases the tool’s upvote count by one.
Expected Result: The tool’s upvote count decreases by one, and the updated count displays.

Test Case 3: Software tools readjust position based on votes

Description: Whenever a software tool surpases another in votes it should swap positions.
Expected Result: The software tools postion adjusted in response to an it surpassing another in rating or vise versa.
